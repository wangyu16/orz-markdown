## Design Context

### Users
Each theme serves a different audience and use case:
- **light-academic**: Academics and researchers — scholarly papers, lab notes, chemistry/math-heavy documents read at a desk
- **beige-decent**: Publishers — polished prose, articles, essays; a refined print-like feel
- **light-playful**: Personal use — personal blogs, journals, casual notes viewed in a relaxed context
- **dark-elegant / light-neat**: General-purpose web rendering — technical documents, software documentation, mixed content

### Brand Personality
No single brand — each theme must have its own strong visual identity. The overarching goal is: **distinctive, varied, and free of generic AI aesthetics**. Anti-references: anything that looks templated, safe, or like it came out of an AI design generator.

### Aesthetic Direction
- Each theme should commit to a clear, specific aesthetic POV — not a "nice default"
- Avoid the AI color palette (cyan-on-dark, purple gradient, neon-on-dark)
- Avoid AI structural tells (side-stripe callouts, gradient text, glassmorphism)
- Color and contrast quality is a top concern — perceptually uniform palettes preferred
- Fonts should be chosen intentionally for each theme's context, not from the reflex list

### Design Principles
1. **Each theme is a distinct object** — they should not look like variations of the same template
2. **Typography as identity** — font choice must match the purpose and feel of the theme
3. **Color serves the content** — no decorative color without functional purpose
4. **Accessibility is non-negotiable** — WCAG AA contrast is the floor, not the ceiling
5. **No AI tells** — if it "looks AI-generated," rewrite it
