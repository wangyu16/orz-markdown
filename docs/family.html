The editable family

Three tools, one renderer. Each turns Markdown into a single, self-contained, editable HTML file — different shapes for different documents.

What they all share

  • One portable file. The Markdown source lives inside; the file renders itself on open.
  • Edit in place. A built-in pop-out editor (CodeMirror) with live preview; Save writes back to the same file (Chromium browsers, via the File System Access API).
  • Copy as Markdown. Select rendered content, copy, get clean Markdown source.
  • Self-updating, opt-in. When a newer framework ships, the editor offers a one-click update from the canonical packages.
  • Self-contained or CDN. --inline embeds the engine + theme (works offline); --cdn references jsDelivr. Fonts, images, and math/diagram libraries load from CDN at view time.

orz-mdhtml  .md.html

A continuous document — the classic article, README, or notes view, rendered in a themed iframe for true WYSIWYG. The editor sits beside a live preview with optional scroll-sync. GitHub

orz-slides  .slides.html

A reveal.js slide deck — present from one file in any browser. A layout grammar (2col, splits, regions), structure templates (title/section/closing), and presenter tools: speaker view, step-reveal fragments, a timer, slide numbers. GitHub

orz-paged  .paged.html

A paginated print document — Markdown laid out as pages with paged.js (A4/Letter, margins, running headers/footers, page numbers). Edit side-by-side, then export to PDF by printing — no PDF binary, no server. Curated templates for articles, letters, CVs, reports, and exams. GitHub
